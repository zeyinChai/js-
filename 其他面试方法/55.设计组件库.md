- 设计组件库我一般会从 组件设计、样式体系、工程化能力、类型支持、可维护性、性能和生态 这几个维度考虑。比如组件 API 要稳定且可组合，样式要支持主题化，工程上要支持 ESM 和按需加载，类型上要提供完善的 TypeScript 声明，同时要有文档系统、测试体系和版本管理，保证组件库可以长期演进。

# 一、组件设计原则（最核心）

1. 首先要保证组件本身是 高内聚、低耦合、可组合的
- 保持 API 稳定性（向后兼容）

2. 组件职责单一
- Table 只负责渲染、数据请求交给业务层，否则组件库会越来越重。

# 二、样式体系设计

1. 主题能力（Theme）
CSS Variables、Less / Sass 变量、Design Token

2. 样式隔离
CSS Modules、BEM规范

# 三、工程化能力

1. 打包策略（umd、esm）

2. 按需加载

3. External 依赖（react、vue）

4. dts支持打包ts类型，不然使用方拿不到组件类型

# 四、可维护性

1. 单元测试
2. 文档系统（storybook、dumi）
3. 发布组件的时候需要 changelog
